### 3.1 User Flow: PC-Configurator

#### 1. Главная страница (Landing)

**Старт → Экран приветствия**
Пользователь может:

- Ознакомиться с концепцией платформы.
- Перейти в каталог готовых сборок.
- Перейти в раздел «О проекте».

**Сценарии:**

- Клик по кнопке «В каталог» → Переход к списку ПК.
- Клик по логотипу → Обновление главной страницы.

#### 2. Каталог сборок

**Пользователь может:**

- Просматривать сетку доступных ПК.
- Видеть краткие характеристики (ID, модель, базовая цена).
- Отслеживать статус сборки (метка «Проверено»).

**Сценарии:**

- Выбор карточки #042 → Переход на страницу Core-сборки.
- Выбор карточки #043 → Переход на страницу Workstation-сборки.
- Пустой каталог (ошибка загрузки) → Сообщение «Сборки временно недоступны».

#### 3. Страница товара (Конфигуратор)

**На экране выбранного ПК пользователь может:**

- Изучить подробный список комплектующих.
- Просмотреть бенчмарки (FPS в играх, время рендера, температуры).
- Сменить ракурс системного блока через миниатюры фото.

#### 4. Кастомизация (Выбор опций)

**Пользователь:**

- Выбирает дополнительные компоненты в выпадающем списке (RAM, Охлаждение).

**Сценарии:**

- Выбор платного апгрейда → Динамическое обновление итоговой цены (JS).
- Выбор базовой версии → Возврат к исходной цене.

#### 5. Связь с мастером (Target Action)

**Пользователь:**

- Нажимает кнопку «Связаться с мастером».

**Сценарии:**

- Успешный клик → Переход по внешней ссылке (TG/WA) или открытие формы.
- Ошибка перехода → Уведомление о технических работах.

#### 6. О проекте (Техническая часть)

**Пользователь может:**

- Посмотреть текущую версию билда.
- Изучить технический стек (HTML, CSS, JS).
- Увидеть статус выполнения модулей проекта.

#### 7. Навигация

**Пользователь может:**

- Нажать «← Назад в каталог» → Возвращение к списку с сохранением позиции скролла.
- Нажать на Логотип → Сброс всех фильтров и возврат на главную.
