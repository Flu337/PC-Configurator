# Техническое задание на разработку сайта «PC-Configurator»

## 1. Введение
Данное техническое задание описывает мое требование по разработке веб-платформы для подбора и реализации кастомных компьютерных сборок. Проект ориентирован на эстетику минимализма и высокую информативность.

## 2. Общие сведения

### 2.1 Назначение системы
Сайт PC-Configurator предназначен для создания профессиональной витрины и экосистемы, где мастера выставляют свои работы, а покупатели находят сбалансированные ПК без торговых наценок ритейла. Основная задача — обеспечить прозрачность сделки и соответствие железа заявленным характеристикам через систему технических аудитов.

### 2.2 Основные конкурентные преимущества компании
* **Монохромная эстетика:** Чистый интерфейс в стиле архитектурных бюро, не отвлекающий от характеристик комплектующих.
* **Технический аудит:** Обязательные отчеты о температурных режимах и стабильности (бенчмарки) для каждой карточки товара.
* **Прямая связь:** Возможность прямого обсуждения конфигурации с автором сборки для индивидуальной настройки.

### 2.3 Целевая аудитория
* **Покупатели:** Геймеры, архитекторы, IT-специалисты (14–45 лет), ценящие надежность и строгий дизайн.
* **Исполнители:** Опытные билдеры и моддинг-студии, нуждающиеся в качественной подаче своего портфолио.

## 3. Структура сайта
На всех страницах в верхней части расположены следующие элементы:
* **Логотип** (черно-белый, ссылка на главную);
* **Меню сайта:** Главная, Каталог, О проекте, Поддержка ;
* **Кнопки поделиться** (TG, VK);

В нижней части расположены:
* **Копирайт**;
* **Контакты** и ссылки на социальные сети (монохромные иконки).

## 4. Описание экранов

### 4.1 Главная

![Главная страница]({6224F9FB-C0D7-4989-9B29-761AFC2682CC}.png)
### 4.1.1 Каталог сборок
Основное пространство для поиска готовых решений от мастеров.
* **Фильтрация:** Минималистичная боковая панель. Фильтры по назначению (Gaming, Work, Render), по бюджету и по ключевым комплектующим (GPU/CPU).
* **Сетка товаров:** Карточки сборок в изометрической проекции. 
* **Элементы карточки:** Индекс сборки (например, ID: 042), цена, основные спеки и статус «Проверено» (если тесты подтверждены модерацией).

### 4.1.2 Страница конфигурации (Product Page)
Экран, на котором пользователь принимает решение о покупке.
* **Блок визуализации:** Высококачественные фото реальной сборки (не рендеры), демонстрирующие кабель-менеджмент и эстетику.
* **Технический паспорт:** Детальная таблица комплектующих с указанием серийных номеров или конкретных ревизий.
* **Интерактивный REPORT:** Графики производительности, полученные в ходе тестов. Показатели FPS в разрешении 2K/4K и температурная карта системы под нагрузкой.
* **Профиль мастера:** Информационный блок об авторе сборки, его рейтинге и возможности прямой связи.

### 4.1.3 Личный кабинет (Dashboard)
Интерфейс для управления заказами и общением.
* **Для покупателя:** Список отслеживаемых сборок («Избранное»), история переписок с мастерами и статус текущих заявок.
* **Для мастера:** Инструментарий для добавления новых конфигураций, загрузки результатов бенчмарков и управления своим портфолио.

### 4.1.4 О проекте (О проекте / FAQ)
* **Контент:** Описание стандартов сборки, правил платформы и юридической информации. Форма обратной связи для решения спорных моментов.

#### 4.1.1.1 Прототип
Визуальное решение: крупный типографический заголовок и описание преимуществ слева, интерактивная форма «Задать вопрос» справа. В нижней части — три контентных блока: **Mobilise**, **Report**, **Consolidate**.

#### 4.1.1.2 Описание
Обязательные элементы:
* **Сетка сборок:** Карточки товаров в строгой черно-белой гамме.
* **Форма связи:** Поля «Имя», «Email», «Вопрос» и кнопка отправки (добавить в будущем).
* **Поведение:** Динамическое изменение контрастности элементов при взаимодействии с курсором.

## 5. Требования к дизайну
* **Стиль:** Строгий минимализм, отсутствие ярких цветов.
* **Цветовое решение:** * **Фон:** Глубокий черный (#000000).
    * **Текст:** Чистый белый (#FFFFFF).
    * **Второстепенные элементы:** Оттенки серого (#888888).
* **Примеры (референсы):**
    * [rog.asus.com](https://rog.asus.com) — визуальный стиль.
    * [pcpartpicker.com](https://pcpartpicker.com) — логика отображения характеристик и карточки товаров.
* **Графика:** Все изображения комплектующих на сайте должны быть выдержаны в единой темной тональности.

## 6. Технические требования
* **Архитектура:** React SPA.
* **Адаптивность:** Корректное отображение на всех типах устройств.
* **Браузеры:**
    * Google Chrome 42+;
    * Safari 9.1+;
    * EDGE 12+;
    * YaBrowser 15+.
* **Разрешения экранов:**
    * Десктоп: 1280 х 768 — 1920 х 1080.
    * Планшет: 1024 х 768.
    * Мобильные: от 360 х 480.
* **Стандарты:** Соблюдение стандартов W3C, оптимизация скорости загрузки (отклик до 200 мс).
